## Summary
Explains why Obsidian is the ideal external memory for the project and how it solves Claude's forgetting problem. Covers the PARA organizational framework and prompt caching benefits. Includes honest guidance on when to use Obsidian vs. Claude's web interface.

## Key Points
- Obsidian: free, local, plain markdown files — perfect for AI to read directly without copy-pasting.
- CLAUDE.md at vault root = Claude's permanent memory across sessions.
- PARA framework: Projects (active code files), Areas (responsibilities), Resources (API docs, guides), Archives (old work).
- Claude can be directed: "Look at the API docs in my Resources folder, update the server code in my Projects folder."
- Prompt caching in local environment: Claude "freezes" project structure, only charges for the new question — up to 70% token reduction.
- Simple path: Claude web Projects + Artifacts works fine for small apps. Obsidian is the "feel like a pro" upgrade.
