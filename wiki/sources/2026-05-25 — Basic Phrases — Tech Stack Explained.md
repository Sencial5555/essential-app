## Summary
Beginner-friendly analogies explaining each layer of the Essential tech stack (HTML, CSS, Tailwind, Node.js, Database) and the GitHub/Vercel deployment workflow. Explains when to push to GitHub.

## Key Points
- HTML = framing/bricks (what's on screen). CSS = paint/decor (how it looks). Tailwind = pre-made decorator kit (shortcut CSS classes).
- Node.js = pipes and wiring behind the drywall — handles API calls, checks Stripe subscriptions.
- Database = filing cabinet — stores user emails and scan counts.
- Vercel: connects to GitHub once; auto-deploys on every push within seconds.
- Push to GitHub at three moments: feature completed, end of weekend session, before risky experiments.
- Claude Code handles git commit/push via plain English command.
